4. Role Permissions Matrix

Admin Full Power

  • Dashboard, All Tasks, New Brief
  • Concept approval (review/reject/revision)
  • Change user roles
  • Manage lookup tables (fabrics, categories)
  • Manage designer codes
  • Full CRUD on samples/salvedge
  • Task soft-delete + backward moves
  • Sampling, Analytics, Team

Coordinator Almost Admin

  • Dashboard, All Tasks, New Brief
  • Cannot approve concepts
  • Cannot change roles
  • Cannot manage lookups
  • Task lifecycle (create, assign, advance)
  • Task soft-delete + backward moves
  • Client CRUD
  • Sampling, Analytics, Team

Designer Submit & Work

  • Dashboard, My Board
  • Claim tasks from open pool
  • Submit concepts for review
  • Upload design files
  • Track qty completed
  • Finalize approved concepts
  • Cannot create briefs
  • Cannot move tasks backward

7. Concept Approval Workflow

Designer submits concept
Title + image + client + designer + priority + file upload
DB trigger: concept_code = C-YYYYMMDD-XXXX, md_planned_date = tomorrow
↓
Pending Admin reviews within +1 day
ConceptDetailDrawer → notes + 3 action buttons
↓
Approve
md_status = 'approved'
designer_planned_date = today + 4
↓
Designer finalizes
designer_actual_date = today
Within +4 days
Request Revision
md_status = 'revision_requested'
Designer can edit & resubmit
↓
Back to Pending
Reject
md_status = 'rejected'
Terminal state
Admin-exclusive: Only admin can approve/reject/revise concepts. Coordinators and designers cannot.
DB triggers: md_actual_date + md_reviewed_at auto-stamped on any verdict.
